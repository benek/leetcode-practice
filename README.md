# LeetCode Practice - @JavierBenek

My Java solutions for Leetcode data structures and algorithms practice problems, organized as a lightweight archive of submitted attempts (attemps are submitted on Neetcode.io).

## Repository Structure

```text
Data Structures & Algorithms/
  <problem-name>/
    submission-0.<ext>   ← first submission
    submission-1.<ext>   ← second submission
    ...
```

Each problem has its own folder. Multiple `submission-N.java` files in the same folder represent different attempts and/or implementations for that problem.

All solution files use the common LeetCode/NeetCode style:

- A `class Solution` entry point.
- No Java package declarations.
- Platform-provided method signatures.
- Platform-provided helper types for linked-list and tree problems, such as `ListNode` and `TreeNode`.

## Using the Solutions

These files are designed to be opened or pasted into a LeetCode/NeetCode-style runner. For local experimentation, compile one solution at a time and provide any imports, helper classes, and test harness code that the online platform normally supplies.

## Notes

- There is no project-level build system or automated test suite yet.
- Folder names are descriptive problem slugs, making it easy to search by problem or topic.
